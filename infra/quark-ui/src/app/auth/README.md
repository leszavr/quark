# Страницы аутентификации

## Обзор

В этом документе описано, как создать страницы аутентификации (login/register) для Quark UI.

## Структура директорий

```
src/app/auth/
├── login/
│   └── page.tsx
├── register/
│   └── page.tsx
└── layout.tsx
```

## Страница входа (Login)

### Компоненты
- Форма входа с полями:
  - Имя пользователя/email
  - Пароль
- Кнопка "Войти"
- Ссылка на регистрацию
- Обработка ошибок
- Индикатор загрузки

### Функциональность
- Валидация полей формы
- Отправка данных на `/api/auth/login`
- Обработка успешного входа:
  - Перенаправление в основное приложение
  - Обновление состояния аутентификации
- Обработка ошибок:
  - Неверные учетные данные
  - Проблемы с сетью

## Страница регистрации (Register)

### Компоненты
- Форма регистрации с полями:
  - Имя пользователя
  - Email
  - Пароль
  - Подтверждение пароля
- Кнопка "Зарегистрироваться"
- Ссылка на вход
- Обработка ошибок
- Индикатор загрузки

### Функциональность
- Валидация полей формы
- Проверка совпадения паролей
- Отправка данных на `/api/auth/register`
- Обработка успешной регистрации:
  - Перенаправление на страницу входа
  - Отображение сообщения об успешной регистрации
- Обработка ошибок:
  - Пользователь уже существует
  - Некорректные данные
  - Проблемы с сетью

## Общие компоненты

### AuthLayout
Общий layout для страниц аутентификации:
- Центрирование контента
- Логотип приложения
- Темная/светлая тема
- Адаптивный дизайн

### AuthForm
Общий компонент формы:
- Поля ввода с валидацией
- Кнопка отправки
- Индикатор загрузки
- Отображение ошибок

## Состояние и управление

### Form State
Использование React хуков для управления состоянием формы:
- `useState` для значений полей
- `useState` для состояния загрузки
- `useState` для ошибок

### Validation
Валидация на стороне клиента:
- Проверка заполненности обязательных полей
- Валидация формата email
- Проверка длины пароля
- Проверка совпадения паролей (для регистрации)

## Интеграция с API

### Auth Service
Использование Auth API Service для:
- `login(credentials)`
- `register(credentials)`

### Error Handling
Обработка ошибок от API:
- Отображение понятных сообщений пользователю
- Подсветка проблемных полей
- Повторная попытка отправки

## Дизайн и UX

### Темы
- Темная тема по умолчанию
- Светлая тема как альтернатива
- Плавные переходы между состояниями

### Адаптивность
- Корректное отображение на мобильных устройствах
- Удобное расположение элементов управления
- Оптимизация для сенсорного ввода

### Анимации
- Плавное появление формы
- Анимация индикатора загрузки
- Эффекты при ошибках
