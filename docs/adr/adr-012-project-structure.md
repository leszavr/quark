# ADR-012: Структура проекта и организация файлов

## Статус

Принято

## Контекст

В проекте Quark наблюдалась несогласованность в структуре файлов:

1. Файлы находились не в соответствующих директориях (например, dialog_example.tsx в корне)
2. Неправильные импорты между модулями
3. Отсутствие четких правил именования и размещения файлов

Это создавало трудности при навигации по проекту и нарушало принципы модульности.

## Решение

1. Переместить файлы в соответствующие директории:
   - Компоненты интерфейса в src/components
   - Сервисы в соответствующие директории сервисов
   - Утилиты в src/lib или аналогичные директории
2. Исправить импорты в файлах на относительные пути или алиасы
3. Создать четкие правила именования и размещения файлов
4. Удалить ненужные/пустые директории

## Реализация

- Файл dialog_example.tsx перемещен в infra/quark-ui/src/components/ui/dialog.tsx
- Удалена директория infra/quark-ui/docs/examples
- Обновлена документация по структуре проекта

## Последствия

### Положительные

- Улучшенная навигация по проекту
- Соответствие принципам модульности
- Устранены потенциальные проблемы с импортами
- Четкая организация файлов

### Отрицательные

- Необходимость обновления ссылок на перемещенные файлы
- Временные затраты на реорганизацию
